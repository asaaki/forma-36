# Accessibility

## Our aim is to design and build an inclusive application which is usable for users of all abilities.

### Intro and categories of disabilities.

- Permanent, Temporary, Situational
- Our goal is to be compliant to WCAG Level A and Level AA
- Building empathy

### Assistive Technologies

- Provide alternative text
- Build for keyboard use only, provide focus indicators etc.
- Provide shortcuts and skip links

### Low vision

- Use contrasts ratio 4.5:1 for normal text and 3:1 for large text
- https://webaim.org/resources/contrastchecker/

### Automated Testing

- Describe what to take care of when adding new components

### WCAG 2.1 Level A and Level AA

- Create a Level AA Checklist
